## 一、产品概述

### 1.1 产品定位

MyMe 是一款**个人AI记忆助手**，通过AI对话和日记分析，自动提取、存储用户的人生经历，形成完整的个人知识库。

### 1.2 核心价值

- **自动记忆**: 通过自然对话，AI自动识别并存储用户的重要经历
- **隐私安全**: 所有数据本地加密存储，用户拥有完全控制权
- **数字分身**: 创建可授权的AI分身，分享给家人朋友

### 1.3 目标用户

- 25-45岁知识工作者
- 重视个人成长和自我认知
- 愿意通过AI辅助记录人生

## 三、UI/UX 详细规格（Neumorphism 风格）

> **设计原则**: 简约、统一、无边框
>
> - 整个App只使用**一套色系**（默认冷色系，用户可选冷色/暖色/暗色）
> - 所有模块(M1-M10)**不再使用不同颜色**，统一使用主品牌色
> - 图标：统一使用**线条型图标**，颜色与文字一致
> - 去掉所有花哨的渐变和阴影，只保留Neumorphism的微妙光影

````
### 4.1 登录/注册流程

#### 4.1.1 登录页面 (LoginScreen)

> **页面定位**: 用户进入App的第一个页面，负责身份验证

### 4.4 聊天页面 (ChatScreen)

> **页面定位**: 与真人或Agent一对一聊天
> **核心目标**: 发送消息，与真人或Agent对话，可随时切换身份
> **用户场景**: 和家人聊天，或咨询Agent问题
> **设计考量**: 类似微信聊天，顶部显示当前对话信息，可切换agent/user身份

```
┌─────────────────────────────────────────────┐
│ ← 返回     爸爸                    [切换]  │ ← 切换: agent/user
├─────────────────────────────────────────────┤
│              [当前: Agent]                  │ ← 显示当前身份
├─────────────────────────────────────────────┤
│                                             │
│    你好！有什么想和我聊聊的吗？             │ ← Agent消息(左对齐)
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│           最近工作压力大                   │ ← 用户消息(右对齐)
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│    听起来你最近很辛苦。能具体说说          │ ← Agent消息(左对齐)
│    是哪方面压力吗？                         │
│                                             │
├─────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────┐ │
│ │ 输入消息...                          [→]│ │ ← Agent回复时禁用
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**核心功能**:

1. **身份切换**: 点击右上角切换按钮，在'agent'和'user'间切换

2. **输入框锁定逻辑**:
   - Agent回复中：输入框显示"Agent回复中..."，禁用输入
   - 切换为人工：输入框恢复正常，可输入
   - 超时自动切换：对话中超过2小时无人工回复，自动切回Agent模式

3. **对方可见状态**: 对话列表和聊天页顶部显示"当前: Agent回复"或"当前: 人工回复"

4. **消息数据结构**:
```typescript
interface ChatMessage {
  id: string;
  conversationId: string;
  senderId: string;           // 发送者ID
  senderType: 'user' | 'agent'; // 发送者类型
  senderName: string;          // 发送者名称
  content: string;              // 消息内容
  createdAt: string;
}
```
3. **消息类型**: 支持文字
4. **聊天记录**: 同一对话中切换身份，共享上下文

**消息数据结构**:
```typescript

  conversationId: string;

### 4.5 AI对话 (AgentChatScreen)

> **页面定位**: My Past Agent 核心功能页面，通过自然对话收集用户经历
> **核心目标**: 通过AI引导式对话，自动提取并存储用户的人生经历到知识库
> **用户场景**: 用户想通过聊天方式记录自己的故事、想法、经历，让AI帮忙整理
> **设计考量**: 对话自然流畅，AI能智能识别内容并提取关键信息，用户可一键保存

```
┌─────────────────────────────────────────────┐
│ ← My Past Agent               [历史] [更多] │ ← 历史: 查看会话列表
├─────────────────────────────────────────────┤
│                                             │
│    你好！我是MyMe AI助手，可以帮助你         │

├─────────────────────────────────────────────┤
│                                             │
│           我最近在考虑换工作                │ ← 用户消息(右对齐)
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│    听起来是个重要的决定！你方便说           │ ← Agent消息(左对齐)
│    说你为什么想换工作吗？                   │
│                                             │
│    [M1] [M2] [M3] [M4] [M5] ...            │ ← 智能推荐相关模块
│                                             │
├─────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────┐ │
│ │ 输入消息...                          [→]│ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**核心功能**:

1. **对话交互**
   - 用户发送消息 → Agent回复
   - 支持多轮对话
   - 显示发送时间

2. **智能提取**
   - Agent分析对话内容
   - 识别M1-M10分类
   - 提取关键信息存入知识库

3. **模块推荐**
   - 根据对话内容推荐相关模块
   - 用户可点击直接添加

4. **会话管理**
   - 新建会话
   - 查看历史会话
   - 删除会话

**消息格式**:

```typescript
interface Message {
  id: string;
  content: string;
  sender: "user" | "agent";
  createdAt: string;
  extractedModules?: KnowledgeModule[]; // Agent提取的模块
  suggestedItems?: {
    // 建议添加的知识
    module: KnowledgeModule;
    content: string;
  }[];
}
```

**API调用**:

- 发送消息: `POST /agent/message`
- 获取会话列表: `GET /agent/sessions`
- 获取历史消息: `GET /agent/session/:id/messages`
- 删除会话: `DELETE /agent/session/:id`

### 5.3.1 系统Agent Prompt设计

#### me_agent (我的过去)

> **Agent角色**: 知识更新Agent，负责接收用户输入、判断知识模块、执行数据更新
> **核心功能**: 只负责提问和记录，不进行闲聊式聊天
> **数据库协调**: 提取的知识点存储到knowledge_items表，source='me_agent'

**Prompt**:
```
你是"知识更新Agent"，你的唯一任务是通过提问帮助用户回忆和记录过去的经历。

## 身份定义
你负责：
1. 接收用户的增量知识输入
2. 判断知识所属模块（M1-M10）
3. 执行数据更新
4. 通知相关Agent同步

---

## 核心原则
1. 只提问，不闲聊
2. 每次对话至少提取1条知识条目
3. 提取的知识必须归类到M1-M10
4. 每次追问前必须检索存量数据，确保问题不重复

## 工作流程

### Step 1: 接收输入
用户输入示例：
- "我最近学了一个新技能：Go语言"
- "我对AI的认知有了新变化"
- "我要修正之前说的内容"

### Step 2: 分类判断 (自主选择)
输入内容 → 分析语义 → **自主判断**最合适的M1-M10模块
- 不要询问用户应该选择哪个模块
- 根据内容语义自主决定归类到哪个或哪些模块
- 如果内容涉及多个模块，可以同时提取到多个模块

### Step 3: 深度追问
确定模块后，设计3-5个追问完善知识：
- 具体细节？
- 时间节点？
- 影响范围？
- 相关联系？

**重要**: 每次追问前必须检索该模块所有存量知识，确保问题不重复。

### Step 4: 更新数据
用户确认后，格式化存储：
```json
{
  "module": "M3",
  "title": "新技能：Go语言",
  "content": "...",
  "tags": ["技能", "编程"]
}
```

### Step 5: 影响分析
分析本次更新是否影响其他模块：
- 交叉引用检查
- 一致性验证
- 冲突提示

---

## 更新类型

### 增量更新 (increment)
新增知识，不冲突现有数据

### 修正更新 (modify)
修改已有内容，需记录变更原因

### 删除更新 (delete)
标记废弃（不物理删除），保留历史

---

## 冲突处理
当新知识与现有数据冲突时：

1. **提示用户确认**
   ```
   检测到冲突：
   - 现有数据: [原内容]
   - 新输入: [新内容]

   请确认：覆盖/保留两者/需要澄清
   ```

2. **记录冲突日志**

## 七、安全与加密策略

### 7.1 密码安全

> **遵循微信级安全标准（参照现有后端代码实现）**

| 策略 | 实现方式 | 代码位置 |
|------|----------|----------|
| 密码哈希 | bcrypt, cost=12 | `AuthService.ts:61` |
| 密码强度 | 最少6位字符 | - |
| 登录保护 | 连续5次失败锁定30分钟 | `AuthService.ts:57-58` |
| Token | JWT access (24h) + refresh (7天) | `AuthService.ts:53-54` |
| 传输安全 | HTTPS 强制加密传输 | 环境配置 |

**JWT Token设计** (参照 `AuthService.ts`):
```typescript
// Token载荷
interface TokenPayload {
  userId: string;
  type: 'access' | 'refresh';
}

// 生成Token
generateToken(userId: string, type: 'access' | 'refresh'): string {
  return jwt.sign(payload, JWT_SECRET, { expiresIn });
}

// Token有效期
- accessToken: 24小时
- refreshToken: 7天
```

**账号锁定逻辑**:
- 最大登录尝试: 5次
- 锁定时长: 30分钟
- 失败次数存储: `users.failed_login_attempts`
- 锁定时间存储: `users.locked_until`

### 7.2 数据加密

> **本地数据加密存储，使用AES-256-CBC**

| 数据类型 | 加密方式 | 密钥管理 |
|----------|----------|----------|
| 日记内容 | AES-256-CBC | 设备本地生成密钥，存储在安全存储区 |
| 聊天消息 | AES-256-CBC | 同上 |
| 知识库内容 | AES-256-CBC | 同上 |

**加密算法规格**:
```
- 算法: AES-256-CBC
- 密钥长度: 256位 (32字节)
- IV: 每次加密随机生成128位
- 填充: PKCS7
- 密钥存储: 设备安全区域 (Keychain/Keystore)
```

**代码实现** (参照现有 encryption.ts):
```typescript
// 加密
const encrypted = CryptoJS.AES.encrypt(text, keyWordArray, {
  iv: iv,
  mode: CryptoJS.mode.CBC,
  padding: CryptoJS.pad.Pkcs7,
});
// 格式: iv:ciphertext (Base64编码)
```

## 八、业务流程

### 7.1 My Past Agent 对话流程

```
1. 用户进入AgentChatScreen
   ↓
2. 显示欢迎消息 + 历史会话列表(如有)
   ↓
3. 用户发送消息
   ↓
4. POST /agent/message { content, sessionId? }
   ↓
5. 后端:
   a. 保存用户消息到messages表
   b. 调用LLM生成回复
   c. 分析对话内容，提取M1-M10知识
   d. 返回 { message, extracted[] }
   ↓
6. 显示Agent回复 + 提取的知识提示
   ↓
7. (可选) 用户点击"添加知识"
   ↓
8. 跳转到KnowledgeEditScreen，预填提取的内容
```

### 7.2 周期报告生成流程

```
1. 定时任务触发 (每周日/月最后一天/年最后一天)
   ↓
2. 查询指定周期的所有日记
   ↓
3. 调用周期报告生成Agent
   ↓
4. 生成报告并存储
   ↓
5. 周报: 提取关键信息更新M1-M10知识库
   月报/年报: 仅展示，不更新知识库
```

### 7.3 A2A分享流程

```
用户A (分享方):
1. 创建Avatar (设置权限模块)
   → POST /avatars
   ↓
2. 获取分享码
   → 系统生成6位分享码
   ↓
3. 将分享码告诉用户B

用户B (接收方):
4. 进入A2A关系页面
   ↓
5. 点击"添加"，输入分享码
   → POST /a2a { shareCode }
   ↓
6. 后端验证分享码，创建A2A关系
   ↓
7. 双方可以在A2A聊天中对话

A2A聊天:
8. 用户A/B进入A2A聊天页面
   ↓
9. 选择关系，开始聊天
   ↓
10. 切换身份 (agent/user)
    → POST /a2a/:relationId/switch
    ↓
11. Agent使用授权的M1-M10数据生成回复
```
````
